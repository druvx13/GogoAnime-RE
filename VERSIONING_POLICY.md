# Versioning Policy

**OFFICIAL DOCUMENTATION**

## 1. Scheme
This project adheres to [Semantic Versioning 2.0.0](https://semver.org/).

**Format:** `MAJOR.MINOR.PATCH`

-   **MAJOR:** Incompatible API changes or massive architectural rewrites.
-   **MINOR:** Functionality added in a backwards-compatible manner.
-   **PATCH:** Backwards-compatible bug fixes.

## 2. Release Cycle
-   Releases are tagged in the Git repository.
-   There is no fixed schedule; releases occur when significant features or fixes are ready.

## 3. Deprecation Policy
-   Features to be removed will be marked as deprecated in the MINOR release prior to removal.
-   Removal occurs in the next MAJOR release.
