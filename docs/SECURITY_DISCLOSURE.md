# Security Disclosure Policy

**OFFICIAL DOCUMENTATION**

## 1. Reporting Vulnerabilities
We take the security of this system seriously. If you discover a vulnerability, please report it responsibly.

**Do not open public GitHub issues for security vulnerabilities.**

Please email the repository maintainers directly or use the "Report a Vulnerability" feature on GitHub if enabled.

## 2. Information to Include
-   Type of vulnerability (e.g., SQL Injection, XSS, CSRF).
-   Full path to the affected file.
-   Steps to reproduce.
-   Proof of Concept (PoC) code or screenshots.

## 3. Policy
-   We ask for a reasonable time to fix the issue before public disclosure (typically 90 days).
-   We will not pursue legal action against researchers who find and report security flaws in good faith and in accordance with this policy.

## 4. Supported Versions
Only the latest version of the `main` branch is currently supported for security updates.
